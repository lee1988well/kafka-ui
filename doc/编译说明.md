# kafka-ui 编译说明

## 环境要求

| 工具 | 版本要求 | 说明 |
|------|---------|------|
| Java | 17 | 用于编译后端代码 |
| Maven | 内置 `./mvnw` | 项目自带 Maven Wrapper，无需单独安装 |
| Node.js | 无需手动安装 | 由 `frontend-maven-plugin` 在构建时自动下载 v18.17.1 |
| pnpm | 无需手动安装 | 同上，由插件自动管理 |

> 本机 Java 路径示例：`/opt/homebrew/opt/openjdk@17/bin`，如未在 PATH 中，可在执行命令前设置：
> ```bash
> export JAVA_HOME=/opt/homebrew/opt/openjdk@17
> export PATH=$JAVA_HOME/bin:$PATH
> ```

---

## 构建命令

在**项目根目录**（即 `kafka-ui/` 目录，`pom.xml` 和 `mvnw` 所在的目录）下执行：

```bash
cd /path/to/kafka-ui
./mvnw clean package -pl kafka-ui-api -am -Pprod -Ddocker.skip=true -DskipTests
```

参数说明：

| 参数 | 说明 |
|------|------|
| `clean` | 清理上次构建产物 |
| `package` | 执行编译、打包 |
| `-pl kafka-ui-api` | 只构建 `kafka-ui-api` 模块 |
| `-am` | 同时构建 `kafka-ui-api` 所依赖的模块（`kafka-ui-contract`、`kafka-ui-serde-api`） |
| `-Pprod` | 启用 `prod` Profile，触发前端构建（pnpm install + pnpm build）并将前端产物打入 JAR |
| `-Ddocker.skip=true` | 跳过 Docker 镜像构建步骤（`-Pprod` 会触发 `fabric8` 插件尝试构建镜像，本地无 Docker 守护进程时需要跳过） |
| `-DskipTests` | 跳过单元测试，加快构建速度 |

## 构建过程说明

构建分为以下阶段，总耗时约 30 秒：

1. **kafka-ui-contract**（约 9 秒）：根据 OpenAPI 规范生成 Java 和 TypeScript API 接口代码
2. **kafka-ui-serde-api**（约 1 秒）：编译序列化/反序列化插件 API
3. **kafka-ui-api**（约 20 秒）：
   - 下载并安装 Node.js v18.17.1（首次构建，后续有缓存）
   - 执行 `pnpm install` 安装前端依赖
   - 执行 `pnpm build` 构建 React 前端
   - 将前端产物复制到 `src/main/resources/static/`
   - 编译 Java 后端代码
   - 打包成 Spring Boot 可执行 JAR

## 构建产物

```
kafka-ui-api/target/kafka-ui-api-0.0.1-SNAPSHOT.jar
```

该 JAR 包含：
- Spring Boot 后端
- React 前端静态资源（`BOOT-INF/classes/static/`）
- 所有依赖库

## 常见问题

### 问题一：构建后前端页面报错（`static/index.html not found`）

**原因**：执行构建时未加 `-Pprod`，导致前端未被编译打入 JAR。

**解决**：必须使用 `-Pprod` 参数，确保 `frontend-maven-plugin` 执行前端构建流程。

---

### 问题二：构建时提示 Node 版本不匹配

**原因**：系统安装的 Node 版本与项目要求（v18.17.1）不符。

**解决**：无需处理。`-Pprod` 模式下，`frontend-maven-plugin` 会自动下载并使用项目指定版本的 Node，与系统 Node 无关。

---

### 问题三：构建时 Docker 镜像构建失败

**原因**：`-Pprod` 会激活 `fabric8 docker-maven-plugin`，尝试在本地构建 Docker 镜像。

**解决**：加上 `-Ddocker.skip=true` 参数跳过该步骤。

---

## 部署到容器

构建完成后，将 JAR 替换到运行中的 Docker 容器：

```bash
# 查看容器 ID
docker ps

# 复制 JAR 到容器（替换 <容器ID>）
docker cp kafka-ui-api/target/kafka-ui-api-0.0.1-SNAPSHOT.jar \
  <容器ID>:/app/kafka-ui-api-0.0.1-SNAPSHOT.jar

# 重启容器使修改生效
docker restart <容器ID>
```
